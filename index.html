virtual machine

  • full os in isolation
  • fairly big overhead
  • performance impact

containers

  • full process isolation
  • no big overhead
  • no performance impact
  • FreeBSD Jails
  • Solaris Zones
  • OpenVZ
  • Linux Containers

Linux container support

  • cgroups: limits, accounts for, and isolates the resource usage
  • namespaces: isolates and virtualizes system resources of a collection of processes
  • capabilities: fine-grained control over superuser permissions
  • made lxc easy
  • then own implementation in libcontainer
  • windows: hypervisor, (native support?)
  • macOS: hypervisor
  • uses its own 'lan'
  • not default exposed
  • changes firewall settings
  • links containers together via hosts
  • physical machine
  • other containers
  • remote machine
  • fat container
  • tried to mimic vm
  • hard to see what's going on
  • separate containers
  • volumes
  • linked nicely
  • using docker-compose
  • allows easy configuration
  • simplifies volumes
  • simplifies the linking
  • simplifies networking
  • permission problems
  • containers processes do not have same uid/gid
  • usually a linux only problem
  • other platforms have hypervisor and vm sharing
  • ip adresses used by docker containers are not guaranteed
  • dns resolving made easy for developers
  • docker-machine on osx (slow in our case because vboxshare)
  • heard docker 'native' is also slow

  • to reduce the burden of knowing everything
